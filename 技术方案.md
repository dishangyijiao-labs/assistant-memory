# IDE-Memory 项目技术方案

## 项目概述

IDE-Memory 是一个用于统一管理和记录 IDE 中与 AI 聊天工具会话内容的应用程序。它支持记录和管理 GitHub Copilot、Cursor、VS Code、Cloud Code、CodeX 和 Gemini 等 AI 聊天工具的聊天历史和会话信息。

## 市场调研分析

### 现有解决方案

通过市场调研，我发现了一些类似功能的工具：

1. **Cursor View** (saharmor/cursor-view) - Python + Flask + SQLite 技术栈，用于查看和导出 Cursor AI 聊天历史
2. **Copilot History Keeper** (merijnwip/copilot-history-keeper) - Python 工具，用于迁移和保存 VS Code 中 GitHub Copilot 的聊天历史
3. **Copilot History Viewer** (keithwongg/copilot-history-viewer) - 查看 Copilot 聊天历史的工具
4. **Cursor History** (S2thend/cursor-history) - 浏览、搜索和导出 Cursor AI 聊天历史的工具

### 现有方案的局限性

- 大多数工具只支持单一 AI 工具（如仅支持 Cursor 或仅支持 Copilot）
- 没有一个统一的解决方案支持所有主流 IDE 和 AI 聊天工具
- 缺乏云同步和跨设备访问功能
- 数据存储和管理方式较为简单

## 功能需求

### 核心功能

1. **多 IDE 支持**
   - VS Code 及其扩展（GitHub Copilot、CodeX、Cloud Code）
   - Cursor（基于 VS Code 的 AI 编程工具）
   - 其他支持的 IDE（如 GitHub Codespaces）

2. **多 AI 工具支持**
   - GitHub Copilot Chat
   - Cursor AI
   - VS Code 内置 AI（如 GitHub Copilot）
   - Cloud Code AI（Google Cloud 的 AI 编程工具）
   - CodeX（OpenAI 的代码生成工具）
   - Google Gemini（AI 编程助手）

3. **会话管理**
   - 记录完整的聊天会话（包括提问、回答、代码示例）
   - 支持会话的搜索和分类
   - 导出会话（JSON、Markdown、HTML 格式）
   - 会话标签和分类管理

4. **数据存储**
   - 本地数据库存储（SQLite）
   - 支持数据库备份和恢复
   - 数据加密和安全保护

5. **用户界面**
   - 现代化的 Web 界面
   - 响应式设计，支持多设备
   - 搜索和过滤功能
   - 会话预览和详细查看

## 技术架构

### 技术选型

#### 后端技术
- **语言**: Python 3.8+
- **Web 框架**: Flask 或 FastAPI（轻量级、灵活）
- **数据库**: SQLite（本地存储）
- **ORM**: SQLAlchemy（数据库操作）
- **API 设计**: RESTful API
- **身份认证**: JWT（可选，用于多用户支持）

#### 前端技术
- **框架**: React
- **UI 组件库**: shadcn/ui + Tailwind CSS（现代化、美观）
- **状态管理**: Context API 或 Redux（简单场景）
- **构建工具**: Vite（快速、现代化）
- **图表/可视化**: Recharts 或 Chart.js（数据分析）

#### 系统集成
- **VS Code 扩展**: TypeScript + VS Code API
- **Cursor 集成**: 监听 Cursor 的聊天历史存储
- **数据导入**: 支持从现有工具导入历史数据

## 数据模型设计

### 核心数据表

```sql
-- 用户表（可选，用于多用户支持）
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 会话表
CREATE TABLE sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    tool_name TEXT NOT NULL,
    tool_version TEXT,
    workspace_path TEXT,
    project_name TEXT,
    session_type TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 消息表
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER,
    role TEXT NOT NULL, -- user/assistant
    content TEXT NOT NULL,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

-- 代码示例表（用于存储代码片段）
CREATE TABLE code_snippets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    message_id INTEGER,
    code TEXT,
    language TEXT,
    filename TEXT,
    FOREIGN KEY (message_id) REFERENCES messages(id)
);

-- 会话标签表
CREATE TABLE session_tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER,
    tag TEXT,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

-- 配置表
CREATE TABLE settings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    key TEXT,
    value TEXT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 关键技术挑战

### 1. 多工具集成

不同 AI 工具的聊天历史存储方式可能有所不同，需要统一的接口来解析和存储数据。

### 2. 数据兼容性

需要处理不同 AI 工具产生的不同格式的聊天数据，包括文本、代码片段、图片等。

### 3. 性能优化

当聊天历史非常大时，需要优化查询和展示性能，实现分页和懒加载。

### 4. 数据同步

实现跨设备和跨工具的数据同步，确保用户的数据在不同设备上保持一致。

## 技术方案优势

### 1. 统一管理

提供一个统一的平台来管理所有 IDE 和 AI 工具的聊天历史。

### 2. 数据分析

提供数据分析功能，帮助用户了解聊天模式和习惯。

### 3. 可扩展性

架构设计灵活，便于添加新的 IDE 和 AI 工具支持。

### 4. 安全性

本地存储确保数据安全，避免将敏感信息发送到外部服务器。

### 3. 开发流程

1. 克隆仓库
2. 安装后端依赖：`pip install -r requirements.txt`
3. 安装前端依赖：`cd frontend && npm install`
4. 创建数据库：`python backend/manage.py db upgrade`
5. 启动开发服务器：`python backend/app.py`
6. 启动前端开发服务器：`cd frontend && npm run dev`
